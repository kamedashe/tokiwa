# TokiWa

Трекер просмотра аниме: каталог, списки со статусами, прогресс по сериям и
подсчёт того, сколько времени этот список на самом деле займёт.

## Стек

| Слой | Решение |
| --- | --- |
| Фронтенд | Next.js 15 (App Router), React 19, Tailwind CSS v4 |
| Данные | Prisma + PostgreSQL (Neon или Supabase) |
| Аккаунты | Auth.js v5, вход через Discord |
| Метаданные | Jikan (MyAnimeList), Shikimori, AniList |

Анимации сделаны нативно — IntersectionObserver для появления секций и
`mousemove` для параллакса в шапке. Framer Motion не подключён: на этих двух
эффектах он весит больше, чем даёт.

## Почему это трекер, а не смотрелка

Изначально задумывалась смотрелка со встроенным плеером. От неё отказались,
и вот почему:

Сервисы с русским дубляжом (Кинопоиск, Okko, Wink, Иви) держат контент под DRM
и встраивать себя не дают — публичного API для эмбеда у них нет. Сервисы,
которые эмбеды дают (YouTube, RUTUBE, VK), официального аниме почти не
содержат, а то, что есть у лицензионных каналов вроде Muse Asia, закрыто
региональными блокировками для СНГ.

Остаются пиратские источники — а это блокировки, нестабильные ссылки и
претензии правообладателей.

Поэтому плеера нет. Вместо него — блок «Где посмотреть» со ссылками на
лицензионные сервисы, подобранные под страну посетителя. Всё остальное
(каталог, поиск, списки, прогресс) от этого не пострадало.

## Флагманская фича: «Сколько у меня времени»

`/backlog` переводит список из строчек в часы.

- Сколько всего впереди, сколько начато и не закончено, сколько уже посмотрено.
- Во что это превращается при разном темпе: «по 2 часа в день — 6 недель».
- **Подбор по бюджету времени.** Есть вечер, ночь, выходные или отпуск —
  показывает, что из списка реально закрыть целиком за это время.

Расчёт живёт в `src/lib/backlog.ts` (чистые функции, без обращений к БД).
Длительность серии берётся из Jikan; где её нет — подставляется типовая в 24
минуты, и такая оценка помечается значком `≈`. У фильмов `durationMin` — это
длительность целиком, поэтому на число серий она не умножается.

## Запуск

Нужна база PostgreSQL — подойдёт бесплатный тариф [Neon](https://neon.tech)
или [Supabase](https://supabase.com). У Neon есть ветки, удобно завести
отдельную для разработки.

```bash
npm install
cp .env.example .env       # заполнить DATABASE_URL и секреты
npm run db:migrate         # создать таблицы
npm run db:seed            # забрать ~90 тайтлов из Jikan
npm run dev
```

Секреты генерируются так:

```bash
openssl rand -base64 32    # для AUTH_SECRET и SYNC_SECRET
```

### Не собирать прод при работающем dev-сервере

`next build` пишет в тот же каталог `.next`, из которого читает `npm run dev`.
Если запустить сборку, пока крутится дев, тот развалится с ошибками вида
`Cannot find module './vendor-chunks/…'` или `ENOENT … _document.js`.

Для проверки типов есть `npm run typecheck` — он в `.next` не лезет. Если всё
же сломалось: остановить dev, `rm -rf .next`, запустить снова. То же с
`prisma generate` — дев-сервер держит движок, генерация падает с `EPERM`.

## Деплой на Vercel

1. Залить репозиторий на GitHub, в Vercel выбрать **Import Project**.
2. Прописать переменные окружения из `.env.example`. Строк подключения **две**,
   и они отличаются только `-pooler` в имени хоста: `DATABASE_URL` — пулерная,
   по ней ходит приложение; `DIRECT_URL` — прямая, по ней идут миграции.
   Через пулер в transaction mode миграции не отрабатывают, а без пулера
   serverless-функции быстро исчерпают лимит соединений.
3. Добавить `CRON_SECRET` — Vercel сам подставит его в заголовок при вызове
   по расписанию.
4. В Discord Developer Portal добавить боевой redirect URI:
   `https://<домен>/api/auth/callback/discord`.
5. Первый деплой. Миграции накатываются сборкой (`prisma migrate deploy`
   в скрипте `build`), но каталог приедет пустым.
6. Наполнить каталог — разово дёрнуть ручку синхронизации:

```bash
curl -X POST "https://<домен>/api/sync?pages=2" -H "Authorization: Bearer $SYNC_SECRET"
```

Дальше каталог поддерживает себя сам: в `vercel.json` заданы два ночных
задания — достройка франшиз и обновление топа с текущим сезоном. Бесплатный
тариф разрешает ровно два задания и не чаще раза в сутки, в этот лимит и
уложились.

## Структура

```
prisma/schema.prisma       Title / Genre / WatchLink / WatchlistEntry + модели Auth.js
scripts/sync-related.ts    достройка франшиз, запускается вручную или кроном

src/lib/jikan.ts           каталог: троттлинг, ретраи на 429/5xx
src/lib/shikimori.ts       русские названия
src/lib/anilist.ts         баннеры + запасной источник связей
src/lib/sync.ts            upsert в БД, обход франшиз
src/lib/backlog.ts         расчёт времени (чистые функции)
src/lib/services.ts        закрытый список легальных сервисов + валидация ссылок
src/lib/watchlist.ts       серверные экшены списков

src/app/backlog/           «Сколько у меня времени»
src/app/admin/             ссылки на сервисы, только для роли admin
```

## Источники данных

Каждый закрывает свою дыру — по отдельности ни один не даёт всего:

| Источник | Что берём | Почему именно он |
| --- | --- | --- |
| Jikan (MAL) | каталог, жанры, оценки, число серий, длительность | базовые метаданные |
| Shikimori | **русские названия** | ни Jikan, ни AniList их не отдают; ключ тот же malId |
| AniList | баннеры, обложки, **связи между частями франшизы** | у Jikan постер 425×600 и никаких баннеров |

Внешние API дёргаются **только** сидингом и ручкой `/api/sync` — страницы
всегда читают из своей БД.

Jikan заметно нестабилен и пачками отдаёт 504, поэтому синхронизация переживает
падение отдельного источника и сохраняет то, что доехало.

### Качество картинок

Бесплатные источники не дают арта под широкий блок: баннеры AniList бывают и
1266×266, обложки — до 460×650. Растянутый на всю ширину оригинал превращается
в мыло.

Поэтому низкое разрешение уходит в **размытую подложку**, где его не видно, а
поверх ставится постер в своих пропорциях — он выводится меньше натурального
размера и остаётся чётким. Коэффициент «пикселей исходника на точку экрана»
держится около 1.0 вместо 0.47 при растягивании.

### Достройка франшиз

Топ и текущий сезон приносят по одному тайтлу из франшизы: есть «Врата
Штейна», но нет ни фильма, ни продолжений. `syncRelated` обходит связи
и подтягивает недостающее.

```bash
npm run db:related              # 20 тайтлов за прогон
npm run db:related -- 30 60     # больше за раз
npm run db:related -- steins-gate   # конкретный тайтл, без очереди
```

Прогон **инкрементальный**: каждый обработанный тайтл помечается
`relatedSyncedAt`, поэтому скрипт можно прерывать и запускать снова.

Связи берутся у Jikan, но его ручка `/relations` для части тайтлов лежит
неделями (у Врат Штейна — пять попыток подряд с 504). Поэтому при её отказе
подключается AniList.

## Каталог

Поиск, фильтр по жанру, сортировка и постраничность — всё через
query-параметры, состояние живёт в URL и шарится ссылкой:

```
/catalog?q=гинтама
/catalog?genre=action&sort=year&page=2
/genre/[slug]
/seasons/[year]/[season]
```

Поиск идёт по русскому, оригинальному и японскому названию. Везде явно
проставлен `mode: "insensitive"`: в PostgreSQL `LIKE` различает регистр.
Проект начинался на SQLite, где регистр игнорируется, — при переезде поиск
молча перестал бы находить «гинтама» вместо «Гинтама».

Поле поиска в шапке фокусируется по `/`.

## Аккаунты и списки

Auth.js v5, провайдер — Discord: заводится без ревью и без страницы политики
конфиденциальности, в отличие от Google. Запрашиваем только `identify email`.

Сессии хранятся в БД, а не в JWT: всё равно ходим в базу за списками, зато
сессию можно отозвать, удалив строку.

`WatchlistEntry` держит статус (`watching` / `planned` / `completed` /
`dropped`) и прогресс по сериям. Уникальный индекс `(userId, titleId)` не даёт
завести два статуса на один тайтл; удаление пользователя каскадом уносит его
список.

Ряд «Продолжить просмотр» на главной показывает тайтлы со статусом «смотрю»,
свежие сверху. У гостя ряда просто нет.

### Добавить Google вторым провайдером

В `src/auth.ts` дописать в массив `providers`. Понадобится страница Privacy
Policy — Google требует ссылку на неё, чтобы вывести приложение из режима
Testing (там лимит в 100 пользователей, каждого добавляешь руками).

### Рендеринг

Страницы с шапкой динамические: шапка показывает профиль, а главная —
персональный ряд. ISR тут неприменим. Если упрётся в нагрузку — включать PPR
либо оборачивать выборки каталога в `unstable_cache` с тегами.

## Где посмотреть

`WatchLink` не принимает произвольный хост. Админ выбирает сервис из закрытого
списка `SERVICES`, а `checkWatchLink` сверяет, что ссылка ведёт именно на него —
«Кинопоиск», указывающий на сторонний домен, не сохранится.

### Регионы

У каждого сервиса указаны страны: Кинопоиск, Okko, Wink, Иви, START,
PREMIER — `RU`; MEGOGO, sweet.tv, Київстар ТБ, youtv — `UA`; Netflix и
Crunchyroll — везде.

Страна берётся из geo-заголовка платформы (`x-vercel-ip-country`, у
Cloudflare — `cf-ipcountry`). Локально заголовка нет, тогда показываем всё.

Недоступные сервисы не прячем, а опускаем вниз под заголовок «может не
работать в вашей стране» — у людей бывают VPN и подписки других регионов.

Украинский каталог тонкий не случайно: большую часть прав на аниме в Украине
выкупили российские компании. MEGOGO при этом уже выпускает аниме с украинской
озвучкой.

### Партнёрские ссылки

Шаблон на каждый сервис задаётся переменной `AFFILIATE_<SERVICE>` с
плейсхолдером `{url}`:

```
AFFILIATE_OKKO="https://ad.admitad.com/g/xxxxxxxx/?ulp={url}"
```

Шаблон привязан к аккаунту в CPA-сети, поэтому живёт в окружении, а не в коде.
Не задан — ссылка остаётся обычной. Шаблон без `{url}` игнорируется, чтобы не
отдать битый адрес.

Партнёрские ссылки получают `rel="nofollow sponsored"` и видимую пометку
«реклама». Это не украшательство: поисковики требуют `sponsored`, а
немаркированная реклама — нарушение закона (в РФ с 2022 года ещё и
обязательная регистрация в ОРД с получением erid).

## Админка

`/admin` — привязка ссылок на сервисы. Доступна только роли `admin`.

Первый админ назначается через `ADMIN_EMAILS`: емейлы оттуда получают роль при
входе. Иначе назначать было бы некому — в свежей БД пользователей нет.

Посторонним `/admin` отдаёт **404, а не 403** — незачем сообщать, что админка
вообще существует.

Права проверяются в серверных экшенах, а не только на странице: экшены
дёргаются напрямую по HTTP, мимо всякого UI. `requireAdmin()` стоит первой
строкой в каждом.

## Мобильная версия

Навигация вынесена в нижнюю панель: трекером пользуются одной рукой, и до низа
экрана большой палец дотягивается, а до верхнего угла — нет. Учтён
`safe-area-inset`, чтобы панель не налезала на системную полоску айфонов.

Карточки в сетке идут по две в ряд, жанровые фильтры — горизонтальной лентой
вместо простыни переносов.
