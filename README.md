# TokiWa

Аниме-каталог на Next.js. Главная страница собрана по макету
`Главная.dc.html` (вариант **1A — Cinematic**).

## Стек

| Слой | Решение |
| --- | --- |
| Фронтенд | Next.js 15 (App Router), React 19, Tailwind CSS v4 |
| Данные | Prisma + SQLite (локально) → PostgreSQL на Supabase/Neon |
| Метаданные | [Jikan v4](https://docs.api.jikan.moe/) (обёртка над MyAnimeList) |
| Видео | Только ссылки на эмбеды легальных источников, своего хостинга нет |

Анимации (появление секций, параллакс в hero) сделаны нативно —
IntersectionObserver + `mousemove`, ровно как в макете. Framer Motion пока не
нужен: он весит больше, чем даёт на этих двух эффектах.

## Запуск

```bash
npm install
cp .env.example .env
npx prisma db push     # создать dev.db
npm run db:seed        # забрать ~90 тайтлов из Jikan
npm run dev
```

### Не собирать прод при работающем dev-сервере

`next build` пишет в тот же каталог `.next`, из которого читает `npm run dev`.
Если запустить сборку, пока крутится дев, тот развалится с ошибками вида
`Cannot find module './vendor-chunks/…'` или `Cannot find module './331.js'`.

Лечится так:

```bash
# остановить dev-сервер, затем
rm -rf .next
npm run dev
```

То же самое с `prisma generate`: dev-сервер держит `query_engine-windows.dll.node`,
и генерация падает с `EPERM`. Сначала остановить сервер.

## Структура

```
prisma/schema.prisma      Title / Genre / Episode
src/lib/jikan.ts          клиент Jikan: троттлинг 3 req/s + ретраи на 429/5xx
src/lib/sync.ts           upsert в БД, витрина главной
src/lib/queries.ts        выборки для страниц
src/lib/gradient.ts       градиент-заглушка из макета (когда постера нет)
src/components/           hero, карточки, ряды, плеер
src/app/api/              titles, titles/[slug], sync
```

### Кэширование

Внешний API дёргается **только** сидингом и ручкой `POST /api/sync` — страницы
всегда читают из своей БД. Главная ревалидируется раз в час.

```bash
curl -X POST http://localhost:3000/api/sync?pages=2 \
  -H "Authorization: Bearer $SYNC_SECRET"
```

На Vercel это вешается на Cron. Jikan регулярно отдаёт 504 — синхронизация
переживает падение отдельного источника и сохраняет то, что доехало.

### Серии, озвучки и плеер

Своего видео нет: у `Episode` хранится `embedUrl` + `source`. `Player`
оборачивает чужой iframe своим UI — выбор серии, выбор озвучки, рамка,
оверлей загрузки.

**Несколько озвучек на серию.** Уникальность стоит по
`(titleId, number, source)`, поэтому одна серия может лежать несколькими
строками с разными источниками. Плеер группирует их по номеру: кнопка серии
одна, а дорожки выносятся в отдельный переключатель. Выбранная озвучка
запоминается и переносится на следующие серии. Точка на кнопке серии значит,
что дорожек больше одной.

**Чего сделать нельзя.** Выбор качества и переключение аудиодорожек внутри
чужого фрейма недоступны — отдаётся то, что даёт плеер источника. Своё
качество потребовало бы собственного хранилища, транскодинга и CDN, то есть
ровно того, что этот проект сознательно не делает.

### Два способа посмотреть

Расклад на рынке такой: сервисы с русским дубляжом (Кинопоиск, Okko, Wink,
Иви) держат контент под DRM и эмбедов не дают. Сервисы, которые эмбеды дают
(YouTube, RUTUBE, VK), официального аниме содержат мало. Поэтому на странице
тайтла два блока:

- **«Смотреть здесь»** — `Episode` с iframe. Работает для того, что правда
  разрешено встраивать: официальные каналы дистрибьюторов на YouTube
  (Muse Asia, Ani-One, Aniplex).
- **«Где посмотреть»** — `WatchLink` с внешними ссылками на лицензионные
  сервисы.

Блоки независимы: у тайтла может быть и то, и другое, либо что-то одно.

`WatchLink` не принимает произвольный хост. Админ выбирает сервис из закрытого
списка `SERVICES` (`src/lib/services.ts`), а `checkWatchLink` сверяет, что
ссылка ведёт именно на него — «Кинопоиск», указывающий на сторонний домен, не
сохранится.

### Регионы

У каждого сервиса в `SERVICES` указаны страны, где он работает: Кинопоиск,
Okko, Wink, Иви, START, PREMIER — `RU`; MEGOGO, sweet.tv, Київстар ТБ,
youtv — `UA`; Netflix и Crunchyroll — `*`.

Страна берётся из geo-заголовка платформы (`x-vercel-ip-country`, у
Cloudflare — `cf-ipcountry`). Локально заголовка нет, тогда показываем всё.

Недоступные сервисы не прячем, а опускаем вниз под заголовок «может не
работать в вашей стране» — у людей бывают VPN и подписки других регионов.

Украинский каталог тонкий не случайно: большую часть прав на аниме в Украине
выкупили российские компании. MEGOGO при этом уже выпускает аниме с
украинской озвучкой.

### Партнёрские ссылки

Шаблон на каждый сервис задаётся переменной `AFFILIATE_<SERVICE>` с
плейсхолдером `{url}`:

```
AFFILIATE_OKKO="https://ad.admitad.com/g/xxxxxxxx/?ulp={url}"
```

Шаблон привязан к твоему аккаунту в CPA-сети, поэтому живёт в окружении, а не
в коде. Не задан — ссылка остаётся обычной, ничего не ломается. Шаблон без
`{url}` игнорируется, чтобы не отдать битый адрес.

Партнёрские ссылки получают `rel="nofollow sponsored"` и видимую пометку
«реклама». Это не украшательство: поисковики требуют `sponsored`, а
немаркированная реклама — нарушение закона (в РФ с 2022 года ещё и
обязательная регистрация в ОРД с получением erid).

2. `DATABASE_URL` от Supabase/Neon
3. `npx prisma migrate dev`

## Каталог

`/catalog` умеет поиск, фильтр по жанру, сортировку и постраничность —
всё через query-параметры, состояние живёт в URL и шарится ссылкой:

```
/catalog?q=gintama
/catalog?genre=action&sort=year
/catalog?genre=action&page=2
```

Сортировки: `score` (по умолчанию), `year`, `title`. Поле поиска в шапке
фокусируется по `/` и уходит на `/catalog` после паузы в наборе.

Поиск в SQLite регистронезависим только для латиницы (`LIKE` по ASCII).
Для русских названий после переезда на PostgreSQL нужен
`mode: "insensitive"` в `searchTitles`.

## Аккаунты и списки

Auth.js v5, провайдер — Discord (заводится без ревью и без страницы политики
конфиденциальности, в отличие от Google). Сессии хранятся в БД, а не в JWT:
всё равно ходим в базу за списками, зато сессию можно отозвать, удалив строку.

Запрашиваем только `identify email` — ничего не публикуем от имени пользователя.

`WatchlistEntry` держит статус (`watching` / `planned` / `completed` /
`dropped`) и прогресс по сериям. Уникальный индекс `(userId, titleId)` не даёт
завести два статуса на один тайтл; удаление пользователя каскадом уносит его
список.

Ряд «Продолжить просмотр» из макета теперь настоящий: показывает тайтлы со
статусом «смотрю», свежие сверху. У гостя ряда просто нет.

### Добавить Google вторым провайдером

В `src/auth.ts` дописать в массив `providers`. Понадобится страница Privacy
Policy — Google требует ссылку на неё, чтобы вывести приложение из режима
Testing (там лимит в 100 пользователей, каждого добавляешь руками).

### Рендеринг

Страницы с шапкой стали динамическими: шапка показывает профиль, а главная —
персональный ряд. ISR тут больше не применим. Если упрётся в нагрузку —
включать PPR либо оборачивать выборки каталога в `unstable_cache` с тегами.

## Админка

`/admin` — привязка эмбедов к сериям. Доступна только роли `admin`.

Первый админ назначается через `ADMIN_EMAILS`: емейлы оттуда получают роль при
входе. Иначе назначать было бы некому — в свежей БД пользователей нет.

Посторонним `/admin` отдаёт **404, а не 403** — незачем сообщать, что админка
вообще существует.

### Безопасность эмбедов

Значение уходит прямо в `src` у iframe, поэтому `checkEmbedUrl` пропускает
только `https:`. `javascript:`, `data:`, `vbscript:` и обычный `http:`
отбиваются. `EMBED_ALLOWED_HOSTS` дополнительно ограничивает список хостов
(поддомены разрешаются автоматически) — **заполнить перед выкладкой в прод**,
пустое значение означает «любой https».

Проверка живёт в серверных экшенах, а не только в форме: экшены дёргаются
напрямую по HTTP, мимо всякого UI. Права там же — `requireAdmin()` стоит
первой строкой в каждом.

В поле ссылки можно вставить целиком тег `<iframe …>` из «Поделиться» — src
достанется сам, а источник подставится по хосту.

## Навигация по каталогу

- `/genre/[slug]` — тайтлы жанра. Жанры на странице тайтла кликабельны.
- `/seasons` — текущий сезон плюс ссылки на все остальные, свежие вперёд.
- `/seasons/[year]/[season]` — конкретный сезон. Год и сезон валидируются до
  похода в базу; пустой сезон отдаёт 404, а не пустую страницу.

## Прогресс просмотра

Выбор серии в плеере пишет `progress` и ставит статус `watching`. При
следующем открытии тайтла плеер сам встаёт на эту серию. Гостю экшен вернёт
отказ — это штатно, клик просто не сохраняется, логин посреди просмотра не
навязываем.

## Источники данных

Каждый закрывает свою дыру — по отдельности ни один не даёт всего:

| Источник | Что берём | Почему именно он |
| --- | --- | --- |
| Jikan (MAL) | название, описание, жанры, год, оценка, число серий, длительность | базовый каталог |
| Shikimori | **русские названия** | ни Jikan, ни AniList их не отдают; ключ тот же malId |
| AniList | баннеры и обложки высокого разрешения | у Jikan постер 425×600 и никаких баннеров |

### Качество картинок

Бесплатные источники не дают арта под блок 1440×560: баннеры AniList бывают и
1266×266, обложки — до 460×650. Растянутый на всю ширину оригинал
превращается в мыло.

Поэтому в герое низкое разрешение уходит в **размытую подложку**, где его не
видно, а поверх ставится постер в своих пропорциях — он выводится меньше
натурального размера и остаётся чётким. Коэффициент «пикселей исходника на
точку экрана» держится около 1.0 вместо 0.47 при растягивании.

### Достройка франшиз

`/top` и текущий сезон приносят по одному тайтлу из франшизы: есть
«Врата Штейна», но нет ни фильма, ни Zero. `syncRelated` обходит связи
(сиквелы, приквелы, фильмы, спешлы) и подтягивает недостающее.

```bash
npm run db:related            # 20 тайтлов за прогон, максимум 40 новых
npm run db:related -- 30 60   # больше за раз
```

Прогон **инкрементальный**: каждый обработанный тайтл помечается
`relatedSyncedAt`, поэтому скрипт можно прерывать и запускать снова — он
продолжит с места остановки. Внешние API медленные и нестабильные, гонять всё
разом бессмысленно.

То же самое ручкой: `POST /api/sync?mode=related`.

## Наполнение каталога

В админке есть массовый импорт: список ссылок по одной на строку, номера серий
идут подряд от стартового.

Ютубовские ссылки проходят две обработки:

1. **Нормализация в `/embed/`.** `youtube.com/watch?v=…` и `youtu.be/…` во
   фрейме не открываются — YouTube это запрещает. А копируют люди именно их,
   поэтому `toEmbedUrl` приводит любую форму к встраиваемой.
2. **Проверка через oEmbed.** Ключ API не нужен. Ручка отдаёт заголовок и
   канал, а на удалённый, приватный или запрещённый к встраиванию ролик —
   404. Так битая ссылка не попадёт в базу, а название серии подставится само.

Строки обрабатываются независимо: одна кривая не отменяет остальные, а номер
увеличивается только на успешных — дыр в нумерации не будет.

### Про региональные блокировки

Muse Asia и Ani-One Asia — лицензионные каналы, но большинство их роликов
доступно только в Юго-Восточной Азии, Индии и на Ближнем Востоке. Из России
они не проиграются, хотя oEmbed их пропустит: он проверяет существование
ролика, а не право смотреть его из конкретной страны.

Проверять доступность нужно вручную, открыв видео у себя.
